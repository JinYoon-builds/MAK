# MAK 발권 에이전트

터미널과 기차역의 무인발권기를 **말로 부탁하는 응대형 인터페이스**로 바꾸기 위한 음성 기반 키오스크 UI 프로토타입입니다.
노인과 디지털 약자가 복잡한 절차를 직접 따라 누르는 대신, 매표원 캐릭터에게 목적지와 시간을 말하면 AI가 의도를 파악하고 실제 교통편을 조회해 표 선택과 결제 안내까지 도와주는 것을 목표로 합니다.

> 현재 버전은 화면 흐름 검증용 정적 프로토타입입니다. 실제 음성 인식, 결제, 교통편 조회 API는 아직 연결되어 있지 않으며, 마이크 영역과 버튼 클릭으로 인식 결과를 시뮬레이션합니다.

## 최종 목표

MAK는 단순한 버튼형 키오스크가 아니라 **발권 대화를 대신 진행하는 AI 매표원 에이전트**입니다.

사용자는 "부산, 오늘 오후"처럼 자연스럽게 말하고, 시스템은 그 말을 목적지, 날짜, 시간대, 좌석 조건 같은 발권 의도로 바꿉니다. 정보가 부족하면 캐릭터가 "몇 시쯤 출발하실래요?"처럼 되묻고, 충분한 정보가 모이면 실제 열차 데이터를 조회해 후보를 간단히 설명합니다.

정확성이 필요한 열차 시간, 요금, 좌석, 발권 가능 여부는 LLM이 추측하거나 만들어내지 않습니다. 반드시 코레일 등 실제 교통편 API 응답을 기준으로 사용자에게 안내합니다.

## 목표 작동 흐름

1. 화면이 켜지면 매표원 캐릭터가 음성과 큰 글씨로 먼저 인사합니다.
   `어디 가세요? 말씀만 하세요`
2. 사용자가 말합니다.
   예: `부산, 오늘 오후`
3. STT API가 사용자의 음성을 텍스트로 변환합니다.
4. LLM API가 텍스트에서 발권 의도를 추출합니다.
   예: 목적지 `부산`, 날짜 `오늘`, 시간대 `오후`
5. 정보가 부족하면 캐릭터가 되묻습니다.
   예: `몇 시쯤 출발하실래요?`
6. 코레일 열차정보 API가 실제 열차 시간, 요금, 좌석을 조회합니다.
7. 캐릭터가 후보를 큰 글씨와 음성으로 간단하게 안내합니다.
   예: `오후 2시 KTX 있어요. 5만 9천 원이에요. 이걸로 할까요?`
8. 사용자가 확인하면 결제 단계로 넘어갑니다.
9. 캐릭터가 카드 투입구, 표 나오는 위치 등 물리적 행동까지 안내합니다.

## API 역할

| API | 역할 | 비유 |
| --- | --- | --- |
| STT API | 노인의 음성을 텍스트로 변환 | 귀 |
| LLM API | 텍스트에서 의도를 파악하고, 부족한 정보를 되묻고, 자연스러운 응답 문장을 생성 | 뇌 |
| 코레일/교통편 API | 실제 열차 데이터 조회, 요금 확인, 좌석 확인, 발권 요청 | 손 |

LLM은 대화와 의도 파악을 담당하고, 실제 운행 정보와 발권 가능 여부는 교통편 API가 담당합니다.

## 주요 기능

- 9:16 세로형 키오스크 화면을 브라우저 안에서 비율 유지 스케일링
- 시니어 친화 UI: 큰 글자, 명확한 대비, 단순한 선택지, 반복 확인 흐름
- 음성 중심 발권 흐름 시뮬레이션
- AI 매표원 캐릭터를 활용한 안내 화면
- 안전망 화면: 먼저 말 걸기, 자리 비움 확인, 되묻기, 직원 연결 제안
- 프로토타입 검토용 하단 데모 컨트롤

## 화면 흐름

정상 흐름은 7단계입니다.

1. 목적지 듣기
2. 목적지 확인
3. 시간 듣기
4. 요약 확인
5. 표 찾는 중
6. 결과 선택
7. 발권 완료

안전망 화면은 4개입니다.

- 먼저 말 걸기
- 자리 비움 확인
- 되묻기
- 직원 연결 제안

## 파일 구조

```text
.
├── index.html
├── kiosk.css
├── kiosk-app.js
├── assets/
│   ├── mak-character-bust.png
│   └── mak-character-searching.png
└── README.md
```

| 파일 | 역할 |
| --- | --- |
| `index.html` | 키오스크 화면 마크업. 각 화면은 `<section data-screen="...">` 단위로 구성됩니다. |
| `kiosk.css` | 디자인 토큰, 레이아웃, 컴포넌트 스타일, 애니메이션을 관리합니다. |
| `kiosk-app.js` | 화면 전환, 자동 이동 타이머, 자리 비움 카운트다운, 반응형 스케일링을 담당합니다. |
| `assets/` | MAK 매표원 캐릭터 이미지 자산입니다. |

## 실행 방법

별도 빌드 과정이 없습니다. 브라우저에서 `index.html`을 열면 바로 실행됩니다.

로컬 서버로 확인하려면 프로젝트 루트에서 아래 명령을 실행할 수 있습니다.

```bash
python3 -m http.server 8000
```

그다음 브라우저에서 아래 주소를 엽니다.

```text
http://localhost:8000
```

## 데모 조작법

- 화면 중앙의 마이크 영역을 클릭하면 다음 발화 단계로 이동합니다.
- `네, 맞아요`, `아니요`, 표 선택 카드 등 버튼을 클릭해 흐름을 진행합니다.
- 하단 데모 컨트롤의 `‹`, `›` 버튼으로 정상 흐름을 앞뒤로 이동할 수 있습니다.
- 하단 칩 버튼으로 안전망 화면에 바로 진입할 수 있습니다.
- 키보드 `←`, `→` 방향키도 정상 흐름 이동에 사용할 수 있습니다.

하단 데모 컨트롤은 검토용 UI입니다. 실제 키오스크 배포 화면에서는 제거하거나 숨기는 것을 권장합니다.

## GitHub Pages 배포

정적 HTML 프로젝트라 GitHub Pages로 바로 배포할 수 있습니다.

1. GitHub에 새 저장소를 만듭니다.
2. 이 프로젝트를 해당 저장소에 push합니다.
3. 저장소의 `Settings` → `Pages`로 이동합니다.
4. `Build and deployment`에서 `Deploy from a branch`를 선택합니다.
5. `main` 브랜치와 `/root` 폴더를 선택하고 저장합니다.

배포 후 보통 아래 형식의 URL에서 확인할 수 있습니다.

```text
https://<github-username>.github.io/<repository-name>/
```

## 향후 연동 포인트

- 음성 인식: `kiosk-app.js`의 `[data-go]` 클릭 기반 이동을 STT API 결과 처리로 교체
- 의도 추출: LLM API로 목적지, 날짜, 시간대, 인원, 좌석 조건을 구조화하고 누락 정보는 되묻기
- 교통편 조회: `searching` 화면 진입 시 코레일/교통편 API 요청 후 `results` 화면 데이터 렌더링
- 결제 및 발권: `done` 화면 진입 전 결제, 좌석 확정, 발권 장치 제어 API 연결
- 운영 환경: Pretendard CDN 대신 로컬 폰트 파일 사용 검토

## 현재 제한 사항

- 목적지, 시간, 열차 정보는 샘플 데이터로 고정되어 있습니다.
- 실제 음성 인식, 자연어 처리, 결제, 발권 장치 제어는 구현되어 있지 않습니다.
- Pretendard 웹폰트를 CDN에서 불러오므로 오프라인 키오스크 환경에서는 로컬 폰트 참조로 바꿔야 합니다.
